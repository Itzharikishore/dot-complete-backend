üìã API Overview

Base URL: http://localhost:5000

Authentication: Cookie-based JWT (httpOnly) or Bearer token

Default Superuser: admin@admin.com / admin123

üîê Authentication

Requests can be authenticated using either:

  1. Cookie-based JWT: Automatically set after login, included in httpOnly cookies
  2. Bearer Token: Include in Authorization header: Authorization: Bearer <token>

üìÅ Route Structure

/api/auth - Authentication endpoints (register, login, password reset)
/api/child - Child-role endpoints (profiles, activities, progress)
/api/therapist - Therapist-role endpoints (child management, assignments)
/api/admin - Superuser-only endpoints (system management)

üß™ Testing with curl

1. Register User

curl -X POST http://localhost:5000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John Doe",
    "email": "john@example.com",
    "password": "Password@123",
    "role": "child"
  }'

2. Login

curl -X POST http://localhost:5000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "john@example.com",
    "password": "Password@123"
  }'

3. Get Profile (with Bearer token)

curl -X GET http://localhost:5000/api/auth/profile \
  -H "Authorization: Bearer YOUR_TOKEN_HERE"

4. Create Child (Therapist)

curl -X POST http://localhost:5000/api/therapist/children \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN_HERE" \
  -d '{
    "firstName": "Emma",
    "lastName": "Smith",
    "dateOfBirth": "2015-06-15"
  }'

üîë Key Endpoints

Authentication Routes

POST /api/auth/register

Register a new user

Required Fields: name, email, password, role

POST /api/auth/login

Login and receive authentication token

Required Fields: email, password

GET /api/auth/profile

Get current user profile

Auth Required: Yes

Child Routes

POST /api/child/children

Create a child profile

Auth Required: Yes

GET /api/child/children

Get all children

Auth Required: Yes

POST /api/child/progress

Record progress for an activity

Auth Required: Yes

Therapist Routes

POST /api/therapist/activities

Create an activity

Auth Required: Yes (therapist role)

POST /api/therapist/activity-assignments

Assign activity to a child

Auth Required: Yes (therapist role)

Admin Routes

GET /api/admin/therapists

List all therapists

Auth Required: Yes (superuser only)

POST /api/admin/therapists

Create a new therapist

Auth Required: Yes (superuser only)

üíæ Using Postman

  1. Create a new request
  2. Set method (GET, POST, PUT, DELETE)
  3. Enter endpoint URL (e.g., http://localhost:5000/api/auth/login)
  4. Go to "Body" tab and select "raw" with "JSON" format
  5. Paste the JSON payload
  6. Click "Send"
  7. Copy the received token and use in subsequent requests with Bearer token
